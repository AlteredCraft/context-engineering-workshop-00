# Feature Spec Generator

Produce a feature spec for an existing codebase.

## Workflow

1. Ask the user to describe the feature they want to spec.
2. Sketch a rough draft spec in chat — three short sections: **Feature** (one sentence), **Behavior** (the happy path in 2–3 bullets), **Notes** (anything you'd want to flag for a real spec). Keep it under 150 words total.
3. Stop. Tell the user this is a rough first pass — a fuller spec would scan the codebase first and ask more questions.
